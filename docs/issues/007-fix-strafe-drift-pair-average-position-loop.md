# Issue #007: 平移漂移 —— 配对平均位置环

**Status:** `ready-for-agent`
**Created:** 2026-06-07
**Labels:** bug, motor-control, position-loop, strafe

---

## Problem Statement

Mecanum 四轮小车平移时，后轮（RR/RL）比前轮（FL/FR）快，导致车体在平移过程中产生旋转分量，最终起止位置不一致。串口数据显示后轮编码器增量持续领先前轮，四轮独立位置环无法抑制同向对的速度差异。加入 sync PI 耦合后与本位置环形成双闭环竞争，导致末尾轮子振荡但车不移动。

## Root Cause

### 迭代过程

| 版本 | 方案 | 问题 |
|------|------|------|
| v1（原始） | 开环固定速度 + OR 终止 | 快轮到就停，漂移严重 |
| v2 | 逐轮独立位置环（P/I/D） + AND 终止 | 后轮编码器始终领先前轮，"起止位置不同" |
| v3 | v2 + sync P耦合（kp=0.3） | 增益太弱，拉不回差距 |
| v4 | v2 + sync PI耦合（kp=2.0, ki=0.8） | sync PI 和位置环对同一被控量形成双闭环竞争 → 末尾振荡 |

### 根本原因

平移时 FL↔RR 和 FR↔RL 是**同向对**——两轮方向一致、目标相同。四轮独立位置环下，每轮各自追赶 target，机械响应快的（后轮）永久领先。sync 环试图拉平差距，但同时位置环也试图让慢轮加速追上 target——两个环给出了方向相反的修正力，形成振荡。

## Solution

**同向对配对平均位置环**：同向的两轮共享一个位置环，用配对平均编码器增量作为反馈。

```
平移时 Mecanum 运动学（以 LEFT 为例）：
  FL ← 向后  → RR    同向对 A：FL & RR 共享速度命令
  FR → 向前 ← RL     同向对 B：FR & RL 共享速度命令

对 A：d_avg_A = (d0 + d3) / 2    → pos_out_A = PID(target - d_avg_A)
       speed_fl = speed_rr = -pos_out_A   （两轮同一速度命令）

对 B：d_avg_B = (d1 + d2) / 2    → pos_out_B = PID(target - d_avg_B)
       speed_fr = speed_rl = pos_out_B

终止条件：(d_avg_A >= target) && (d_avg_B >= target)
```

同向对内部的速度差异由 `Motor_ControlWheel` 内 10ms 速度环自行消化——速度环已有 KP+KI+KD，不需要位置层额外干预。

## User Stories

1. 作为小车操作者，我希望平移一段距离后起止位置完全一致（无旋转偏移），以便路径规划准确。
2. 作为调试者，我希望串口看到两对编码器增量（FL+RR 平均、FR+RL 平均）各自达标，且两者同步到达 target。
3. 作为维护者，我希望平移位置环逻辑简单清晰，不引入 sync 耦合这种与位置环竞争的复杂结构。

## Implementation Decisions

### 决策 1：配对平均替代逐轮独立

`Car_StrafeDistance` 从逐轮独立位置环改为同向对配对平均位置环。两对各自维护一个 LPF + 积分 + pos_out。

### 决策 2：同向对共享速度命令

同向对的两轮获得完全相同的速度命令（大小和符号），速度差异交由底层速度环消化。方向由 Mecanum 运动学确定。

### 决策 3：终止条件为两对各自平均达标

`(d_avg_A >= target) && (d_avg_B >= target)`，确保前后轮对都到位。相比于四轮 AND，减少了过度约束。

### 决策 4：删掉 sync 耦合代码

sync PI 耦合是本问题的副作用产物，配对平均方案不需要它。删除 sync_int_pair1、sync_int_pair2、corr_pair1、corr_pair2 全部相关代码。

### 决策 5：位置环状态用局部变量

两对各自维护一组 pos_error_prev 和 pos_integral，函数内局部变量，不改动 `Car_t`。

### 决策 6：串口输出格式调整

输出两对编码器平均和各自的 pos_out，例如：
```
s_pair:%.1f,%.1f  out:%.1f,%.1f
```

## Testing Decisions

### 测试方法

1. **串口日志验证**：观察两对编码器增量平均值是否步调一致，同步到达 target。
2. **实车地面测试**：平移 0.3m 左右各 5 次，观察起止位置是否一致（无旋转偏移）。
3. **末尾行为**：确认无轮子抖动/振荡。

### 好测试的标准

- 平移完成后车体位置在起点正侧方（无前/后偏移分量）
- 多次测试一致
- 末尾无异常振动

## Out of Scope

- 不修改 `Car_RotateAngle` 和 `Car_DriveDistance`
- 不调整 Mecanum 运动学对角线因子
- 不调整底层速度环 PID 参数

## System Context (Zoom-Out)

本轮 fix 是 #006（旋转漂移）方案的纠正和改进，只涉及平移运动原语。

### 三个运动原语的控制策略（进化后）

```
┌─────────────────────────────────────────────────────────────┐
│ Car_DriveDistance（直行）                                    │
│   反馈: d_avg = (d0+d1+d2+d3)/4（四轮平均）                  │
│   输出: 单一 pos_out → 四轮同一速度                          │
│   终止: d_avg >= target                                     │
│   风险: 同 #005，四轮平均绑架慢轮                            │
├─────────────────────────────────────────────────────────────┤
│ Car_RotateAngle（旋转）                                      │
│   反馈: 逐轮独立（FL/FR/RL/RR 各自编码器）                   │
│   输出: 每轮独立 pos_out                                    │
│   终止: 四轮 AND 各自达标                                   │
│   风险: 旋转天然解耦（左右反向），逐轮独立在这里是对的        │
├─────────────────────────────────────────────────────────────┤
│ Car_StrafeDistance（平移）← ★ 本轮目标                       │
│   反馈: 配对平均（对 A: FL+RR, 对 B: FR+RL）                │
│   输出: 每对一个 pos_out → 对内两轮同一速度                   │
│   终止: 两对 AND 各自平均达标                                │
│   ★ 关键：速度环消化对内差异                                 │
└─────────────────────────────────────────────────────────────┘
```

### 方案演化路径

```
v1 开环 ──→ v2 逐轮独立 ──→ v3 sync P ──→ v4 sync PI ──→ v5 配对平均 ★
         ↑___失败___↑  ↑_太弱_↑  ↑__双闭环振荡__↑

核心洞察：
  旋转：左右两侧方向相反 → 四轮天然解耦 → 逐轮独立 ✓
  平移：有同向对（FL↔RR、FR↔RL）→ 四轮绑死 → 逐轮独立会导致
        独立 PID 和 sync 环竞争同一被控量 → 振荡
  方案：同向对共享位置环，对内差异由底层速度环消化
```

### 本轮 fix 只改

- **Motor.c** — `Car_StrafeDistance()` 循环体（从逐轮独立 + sync PI 改为配对平均）
- 不涉及 `Car_RotateAngle`、`Car_DriveDistance`、`main.c`、`Car_t` 结构体
- HITL 验证（#007.2）：地面 LEFT/RIGHT 平移各 5 次

## Further Notes

- 该修复是对 #006（旋转漂移）方案在平移场景下的纠正——逐轮独立适用于旋转，但不适用于平动（平移有同向对）。
- 预期代码比 v4 更简洁（删除 sync 耦合，从 4 组状态 → 2 组状态）。
