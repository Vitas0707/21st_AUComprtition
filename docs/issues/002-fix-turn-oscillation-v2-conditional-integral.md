# Issue #002: 转弯振荡 v2 —— 条件积分 + 位置积分

**Status:** `ready-for-agent`
**Created:** 2026-06-06
**Labels:** bug, motor-control, pid

---

## Problem Statement

Issue #001 修复（积分复位 + 积分上限 200）在 80° 左右仍然剧烈抖动。根因是积分复位阈值 20 counts 太小：总行程 ~1000+ counts，80° 处剩余误差 ~447 counts，远大于 20，速度环积分早已饱和在 ±200。`ki=2.94` 时积分项在每个周期注入 588 到输出增量中，PWM 直接打满。同时位置环无积分项，存在静差，迫使操作员设 125° 来达成 90° 转弯。

## Solution

三管齐下消除积分引起的转弯振荡：

1. **扩大积分复位阈值** —— 从 20 counts 提升到 200 counts，在接近目标前清除速度环积分拖尾
2. **速度环条件积分** —— 仅在速度误差较小时累积积分，大误差时主动清零，防止转向/加减速阶段积分饱和
3. **位置环积分项** —— 新增位置积分项（小增益），消除静差，使设定角度等于实际角度

## User Stories

1. As a 比赛小车操作员，I want 转弯指令 90° 时小车转到 90° 而不是需要手动设 125°，so that 路径编程直观准确
2. As a 比赛小车操作员，I want 负载转弯时不再出现抖动摇摆，so that 比赛路线稳定不丢步数
3. As a 调试工程师，I want 积分有条件的累积而不是无条件增长，so that 速度换向和大幅加减速时不产生积分饱和
4. As a 调试工程师，I want 位置环自带积分项消除静差，so that 不需要通过外部加大角度来补偿
5. As a 系统设计者，I want 速度环积分窗口可配置，so that 不同负载和速度下可以独立调优
6. As a 测试者，I want 直线行驶也在接近目标时复位积分，so that 直行过冲也被抑制
7. As a 测试者，I want 无负载时行为不受影响，so that 改动不引入回归
8. As a 开发者，I want 条件积分逻辑集中在 PID 更新函数中，so that 维护和调参方便

## Implementation Decisions

1. **积分复位阈值扩大**：从 20 提高到 200 编码器计数，覆盖约 20% 行程
2. **速度环条件积分**：新增宏 `SPEED_INTEGRAL_WINDOW`（默认 0.15 m/s），速度误差绝对值大于此窗口时不累积积分且清零；窗口内正常累积
3. **速度积分上限保持 200**：不再从 500 降低（已在 #001 中完成）
4. **位置环积分项**：新增 `POS_KI`（默认 0.005）、`POS_INTEGRAL_LIMIT`（默认 30），在 `Car_t` 中增加 `pos_integral` 字段
5. **位置积分清零时机**：与速度积分复位同步 —— 当位置误差低于复位阈值时，同时清零位置积分
6. **复位逻辑**：`Motor_ResetPIDIntegral` 函数保持不变，位置环清零逻辑不变
7. **不修改 `Car_StrafeDistance`**：其为开环控制，不受积分饱和影响
8. **不修改 `Motor_ControlWheel` 行为**：所有改动仅在 PID 更新函数和位置环循环中

## Testing Decisions

- **良好测试的标准**：带负载执行 90°/180°/370° 转弯，通过编码器日志确认无反向转动，设定角度与到达角度误差 < 5°
- **测试方式**：硬件在环 —— 实际带负载小板车运行标准路线，蓝牙串口输出编码器日志
- **回归测试**：无负载时转弯行为与改动前一致；直线行驶到达距离准确

## Out of Scope

- 速度环 Kp/Ki/Kd 重新整定
- PWM 前馈值或死区调整
- 位置环 PD 参数（POS_KP/POS_KD）重新整定
- `Car_StrafeDistance` 改造
- 更换位置环为增量式

## Further Notes

- `SPEED_INTEGRAL_WINDOW` 初始设 0.15 m/s。如果转弯时仍轻微振荡，可以进一步缩小窗口；如果直行时加减速太慢，可以适当扩大
- `POS_KI = 0.005` 是一个保守值。如果 90° 转弯仍然有 2-3° 的静差，可逐步增大到 0.01
- 位置积分上限 30 对应最大补偿量约 `30 * POS_KP = 30` PWM，足以补偿摩擦力差异
- 备选方案是反计算积分（back-calculation anti-windup），但在增量式 PID 中实现复杂，条件积分法在当前架构中更直接
